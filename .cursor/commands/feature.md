# 新特性开发

基于 `.plan/` 下规划文档，逐步开发新特性。

## 开发前

1. **查看里程碑**：打开规划文档的「〇、进度里程碑」，确认当前阶段与待办任务
2. **定位任务**：从任务清单中找到下一个 `- [ ]` 项，按编号（如 4.1.6）执行

## 开发中

- **开发服务器**：不要执行 `pnpm run dev`，已由用户自行启动
- **项目规范**：严格按照：kebab-case 命名、eslint、prettier、commitlint、cspell、unit-test
- **同步更新 Demo（可选）**：
    - 每阶段完成需（如 `views/stageN/index.vue`），并在里程碑中填写对应路由
    - 新增路由时在 `router/routes.ts` 中注册


## 开发后

- 完成的任务将 `- [ ]` 改为 `- [x]`
- 子阶段完成时，同步更新里程碑完成度（如 7/16）
- 若本阶段全部完成，更新里程碑：状态改为 `✅ 已完成`，完成度改为 N/N
- 重要实现调整写入规划文档「实现调整」小节，并同步到本命令
