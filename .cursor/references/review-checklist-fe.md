# 前端 / 体验审查清单（Reviewer）

**用途**：执行 `/review` 时 **先加载本文件**，再逐条对照代码或规划；发现项按 **`Critical` / `Major` / `Minor` / `Suggestion`** 分类（与 `review` 命令一致）。

---

## 使用协议

1. **Read** 本清单全文。
2. 理解审查对象（文件/片段/规划）后再逐项检查。
3. 每条违反：**位置（文件/行或章节）** + **级别** + **原因（WHY）** + **可落地修复建议**。
4. 输出结构：`审查范围` → 按级别分组 → `总体评价`；可选 **1–10 分** 与 **Top 3**（若用户需要量化）。

---

## Critical（阻塞合并 / 上线）

- [ ] **安全与隐私**：XSS 风险、敏感数据进 DOM/URL、不安全的 `v-html`/`dangerouslySetInnerHTML`、凭据/Token 暴露
- [ ] **可访问性硬伤**：关键交互无键盘路径、表单无可见 label、纯颜色表达关键状态、焦点陷阱
- [ ] **数据与状态**：明显逻辑错误导致错误数据提交、破坏主流程的竞态（若可见于前端）

## Major（应尽快修复）

- [ ] **响应式**：主要断点下布局断裂、不可滚动或遮挡主操作
- [ ] **性能明显问题**：大列表无虚拟化/分页策略（若规模已大）、明显无意义的重渲染模式（有证据时）
- [ ] **错误与空态**：主路径缺少错误提示或空状态，用户无法恢复
- [ ] **与设计系统偏离**：同一语义控件在项目中用法严重不一致（若无设计系统则记为「与既有模式不一致」）

## Minor（建议排期）

- [ ] **细节 A11y**：`alt` 空泛、冗余 `aria`、对比度边缘不达标
- [ ] **交互打磨**：Hover 导致布局跳动、缺少 `:focus-visible`、动效未尊重 `prefers-reduced-motion`
- [ ] **文案与 i18n**：歧义、术语不一致、硬编码可配置文案（若项目要求 i18n）

## Suggestion（可选优化）

- [ ] **可读性/结构**：可拆分的巨型组件、可提取的重复逻辑
- [ ] **测试**：高风险区域缺少与行为对齐的测试（若项目有测试文化）

## 交付前速查（视觉 / 交互 / 布局 / 无障碍）

可与 ui-ux-pro-max Pre-Delivery 并用；此处保留最小集：

**视觉**

- [ ] 图标优先 SVG，避免用 emoji 充当图标
- [ ] Hover 不引起布局跳动
- [ ] Light/Dark（若支持）正文对比度 ≥ 4.5:1

**交互**

- [ ] 可点击元素指针样式明确（如 `cursor-pointer`）
- [ ] 过渡时长合理（约 150–300ms）
- [ ] 键盘焦点可见

**布局**

- [ ] 375 / 768 / 1024 / 1440 抽查
- [ ] 无意外横向滚动；固定顶栏不挡主操作

**无障碍**

- [ ] 图片 `alt`；表单控件有 label
- [ ] 颜色非唯一信息载体
- [ ] 尊重 `prefers-reduced-motion`
