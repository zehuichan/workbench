## 一、需求增强

### 原始需求

> 根据 re-table-next（基于 Element Plus el-table 的增强表格）重新规划一版，底层替换为 ant-design-vue 的 a-table。

### 增强后需求

**目标**：创建 `ReAntTable` 组件，功能对齐 `ReTableNext`，底层由 Element Plus `el-table` 替换为 ant-design-vue 4.x `a-table`。

**技术约束**：
- ant-design-vue `^4.2.6`（已安装）
- Vue 3 Composition API + `<script setup>` + TypeScript
- 复用 `@vueuse/core`、`async-validator`、`es-toolkit` 等现有依赖
- 样式使用 SCSS，配合 Ant Design Vue CSS 变量体系
- Vite + Tailwind CSS 环境

**范围边界**：
- 对齐 `ReTableNext` 的全部功能特性（导航、编辑、校验、脏数据、热键、行操作、列设置、自适应高度、分页）
- 保持相同的 Composable 架构风格，最大化复用 UI 无关逻辑
- 对外 API（Props / Events / Expose / Slots）保持语义一致，命名适配 ant-design-vue 惯例

**验收标准**：
- 每阶段有独立 Demo 页可验证
- 全部 5 个 Demo（基础 + stage1~4）功能与 ReTableNext 等价
- TypeScript 无类型错误

## 二、技术方案

### 2.1 el-table vs a-table 关键差异分析

| 维度 | Element Plus el-table | Ant Design Vue a-table | 适配策略 |
| ---- | ---- | ---- | ---- |
| **数据属性** | `:data` | `:dataSource` | Props 名映射 |
| **列定义** | `<el-table-column>` 组件式 + `:columns` | `:columns` 配置式（原生） | 直接使用 columns 配置 |
| **列字段名** | `prop` | `dataIndex` | 类型映射 `prop → dataIndex` |
| **列渲染** | `#default` 插槽 + `formatter` | `customRender` 函数 | 统一走 customRender |
| **表头渲染** | `#header` 插槽 | `title` 支持 VNode / slot | customHeaderCell + slots |
| **选择列** | `type: 'selection'` | `rowSelection` prop | Props 层适配 |
| **序号列** | `type: 'index'` | 无内置，用 columns 自定义 | 自动生成序号列 |
| **展开行** | `type: 'expand'` + `#expand` 插槽 | `expandable` prop | Props 层适配 |
| **行类名** | `:row-class-name` 函数 | `rowClassName` 函数 | 签名兼容 |
| **单元格类名** | `:cell-class-name` 函数 | `customCell` 返回 `{ class }` | 适配为 customCell |
| **单元格点击** | `@cell-click` | `customCell` 返回 `{ onClick }` | customCell 注入 |
| **列宽拖拽** | `@header-dragend` | `onResizeColumn` 或 `vue-draggable-resizable` | 使用 `resizable` + `onResizeColumn` |
| **分页** | 外部 `el-pagination` | 内置 `pagination` prop | 使用内置分页 |
| **固定列/头** | `fixed` 属性 | `scroll` prop + 列 `fixed` | 直接使用 |
| **边框** | `:border` | `:bordered` | Props 名映射 |
| **斑马纹** | `:stripe` | 无内置 | CSS 实现 `nth-child(even)` |
| **溢出提示** | `:show-overflow-tooltip` | `ellipsis` 列属性 | 列级别配置 |
| **maxHeight** | `:max-height` | `scroll.y` | 属性映射 |
| **表格实例方法** | `clearSelection()` 等 | 通过 `rowSelection` 受控 | 状态受控化 |

### 2.2 Composable 复用分析

| Composable | 复用度 | 适配要点 |
| ---- | ---- | ---- |
| `use-editable` | **高（90%）** | 逻辑完全通用，无 DOM/UI 依赖 |
| `use-edit-history` | **完全复用** | 纯逻辑 |
| `use-validation` | **高（85%）** | `scrollToFirstError` 中 DOM 选择器需改为 ant 类名 |
| `use-row-options` | **完全复用** | 纯逻辑 |
| `use-dirty-tracking` | **完全复用** | 纯逻辑 |
| `use-hotkey` | **高（90%）** | `createEditorFocuser` 中 CSS 选择器需改 |
| `use-navigation` | **中（60%）** | cell-click 处理方式不同（customCell vs 事件），类名返回方式不同 |
| `use-class-names` | **中（50%）** | el-table 用 cell-class-name 函数，a-table 用 customCell 返回 class |
| `use-column-options` | **高（80%）** | 列结构 `prop → dataIndex`，其余逻辑相同 |
| `use-adaptive` | **高（85%）** | DOM 选择器（`.el-table__body-wrapper` → `.ant-table-body`）需改 |

### 2.4 关键设计决策

#### 决策 1：单元格交互（导航 + 编辑触发）

**方案**：通过 `columns` 配置的 `customCell` 属性注入 `onClick` / `onDblclick` 事件处理器。

```typescript
// 在 use-columns.ts 中为每列生成 customCell
customCell: (record, rowIndex) => ({
  class: getCellClass(rowIndex, colIndex),
  onClick: () => handleCellClick(record, rowIndex, colIndex),
  onDblclick: () => handleCellDblClick(record, rowIndex, colIndex),
})
```

**优势**：无需额外事件监听，声明式，性能好。

#### 决策 2：单元格渲染（展示态/编辑态）

**方案**：通过 `columns` 配置的 `customRender` 属性，渲染 `ReAntTableCell` 组件。

```typescript
// 在 use-columns.ts 中为每列生成 customRender
customRender: ({ text, record, index }) =>
  h(ReAntTableCell, { item: column, scope: { row: record, $index: index } }, slots)
```

**优势**：与现有 Cell 组件架构一致，插槽透传方式相同。

#### 决策 3：选择列处理

**方案**：将 `type: 'selection'` 的列配置转换为 a-table 的 `rowSelection` prop。

```typescript
// 主组件中检测列配置里是否有 selection 类型
const rowSelection = computed(() => {
  const hasSelection = columns.value.some(c => c.type === 'selection')
  if (!hasSelection) return undefined
  return {
    selectedRowKeys: selectedKeys.value,
    onChange: (keys, rows) => { selectedKeys.value = keys },
    // ...
  }
})
```

#### 决策 4：序号列处理

**方案**：将 `type: 'index'` 的列配置转换为普通列，用 `customRender` 渲染行号。

```typescript
{
  title: '#',
  width: 55,
  customRender: ({ index }) => index + 1,
}
```

#### 决策 5：斑马纹实现

**方案**：a-table 无内置 `stripe`，通过 `rowClassName` 为偶数行添加 CSS 类。

```typescript
rowClassName: (record, index) => {
  const classes = []
  if (stripe && index % 2 === 1) classes.push('re-ant-table-row--striped')
  // ... other class logic
  return classes.join(' ')
}
```

#### 决策 6：编辑器适配器

**方案**：将 Element Plus 组件替换为 ant-design-vue 对应组件：

| 类型 | Element Plus | Ant Design Vue |
| ---- | ---- | ---- |
| input | ElInput | AInput |
| input-number | ElInputNumber | AInputNumber |
| select | ElSelectV2 | ASelect |
| date-picker | ElDatePicker | ADatePicker |
| switch | ElSwitch | ASwitch |
| time-picker | ElTimePicker | ATimePicker |

#### 决策 7：列宽拖拽调整

**方案**：使用 ant-design-vue 4.x 原生的列 `resizable` 支持 + `onResizeColumn` 回调。需要安装 `vue-draggable-resizable`，或使用 `components` 自定义 header cell 配合 resize 逻辑。

> 注意：ant-design-vue 官方示例使用 `vue-draggable-resizable` 实现列宽拖拽。作为替代方案，可在后续阶段实现。

### 2.5 风险与缓解

| 风险 | 缓解措施 |
| ---- | ---- |
| a-table 虚拟滚动与自定义渲染的兼容性 | 默认不启用虚拟滚动，大数据场景单独适配 |
| customCell 事件注入导致的事件冒泡问题 | 在事件处理器中合理 stopPropagation |
| ant-design-vue 4.x 编辑组件 v-model 协议差异 | 统一使用 `value` / `onChange` 或 `v-model:value` |
| 列宽拖拽依赖外部库 | 阶段 4 实现，可选功能 |
| 选择列受控状态管理 | 组件内部维护 selectedRowKeys，通过 expose 暴露 |

## 五、类型设计（核心伪代码）

### ReAntTableColumn 类型

```typescript
import type { ColumnType } from 'ant-design-vue/es/table';
import type { Component, VNode } from 'vue';
import type { RuleItem } from 'async-validator';

export type RowData = Record<string, any>;

export interface CellContext<T = RowData> {
  row: T;
  column: ReAntTableColumn<T>;
  value: any;
  rowIndex: number;
  colIndex: number;
}

/**
 * 列配置：继承 ant ColumnType，扩展编辑/校验/渲染能力
 * - dataIndex 对应 el-table 的 prop
 * - title 对应 el-table 的 label
 * - width 保持不变
 */
export interface ReAntTableColumn<T = RowData>
  extends Omit<ColumnType<T>, 'children'> {
  /** 特殊列类型（selection / index / expand） */
  type?: 'selection' | 'index' | 'expand';
  /** 列显隐 */
  hidden?: boolean;
  /** 单元格是否可编辑 */
  editable?: boolean | ((row: T) => boolean);
  /** 编辑态组件 */
  component?: string | Component;
  /** 编辑组件 props */
  componentProps?:
    | Record<string, any>
    | ((row: T, column: ReAntTableColumn<T>) => Record<string, any>);
  /** 校验规则 */
  rules?: RuleItem | RuleItem[];
  /** 是否必填 */
  required?: boolean;
  /** 自定义渲染函数 */
  render?: (ctx: CellContext<T>) => VNode;
  /** 多级表头子列 */
  children?: ReAntTableColumn<T>[];
}
```

### use-columns.ts 核心逻辑（伪代码）

```typescript
export function useColumns(options: {
  columns: Ref<ReAntTableColumn[]>;
  slots: Slots;
  handleCellClick: Function;
  handleCellDblClick: Function;
  getCellClass: Function;
  isEditingCell: Function;
  showOverflowTooltip: Ref<boolean>;
}) {
  const antColumns = computed(() => {
    return options.columns.value
      .filter(col => !col.hidden)
      .map(col => {
        if (col.type === 'index') {
          return { title: col.title ?? '#', width: col.width ?? 55,
            customRender: ({ index }) => index + 1 }
        }
        // selection/expand 在主组件处理

        return {
          ...filterExtendedProps(col),
          ellipsis: options.showOverflowTooltip.value ? { showTitle: false } : undefined,
          customRender: (renderCtx) =>
            h(ReAntTableCell, { item: col, scope: renderCtx }, options.slots),
          customCell: (record, rowIndex) => ({
            class: options.getCellClass(rowIndex, colIndex),
            onClick: (e) => options.handleCellClick(record, rowIndex, colIndex, e),
            onDblclick: (e) => options.handleCellDblClick(record, rowIndex, colIndex, e),
          }),
          children: col.children?.length
            ? processChildren(col.children)
            : undefined,
        }
      })
  })

  return { antColumns }
}
```

## 六、风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
| ---- | ---- | ---- | ---- |
| ant-design-vue customCell/customRender 性能 | 大数据场景卡顿 | 中 | 使用 `shouldCellUpdate` 优化、考虑虚拟表格 |
| v-model 协议差异 | 编辑器组件值不同步 | 低 | ant 4.x 已统一 v-model，但需逐一验证 |
| 列宽拖拽实现复杂度 | 需要额外依赖 | 中 | 阶段 4 处理，可使用 `vue-draggable-resizable` 或 CSS resize |
| a-table 内置分页与外部数据不同步 | 分页状态错乱 | 低 | 使用受控模式，通过 `current`/`pageSize` props 控制 |
| 选择列从声明式到受控式 | API 差异 | 中 | 内部维护 `selectedRowKeys`，通过 expose 暴露操作方法 |
| 斑马纹 CSS 与 ant 虚拟滚动冲突 | 虚拟滚动时序号不连续 | 低 | 非虚拟模式下使用 CSS nth-child，虚拟模式下用 rowClassName |
