## 二、技术选型与设计原则

### 2.1 基座选择

| 方案 | 说明 | 优劣势 |
|------|------|--------|
| **A. el-table-v2** | 使用 Element Plus 虚拟表格 | 与项目栈一致，但 API 与 el-table 不同，扩展受限 |
| **B. 自研虚拟表格** | 参考 stk-table-vue 自研 DOM 虚拟滚动 | 完全可控、无依赖、可深度定制 |

**决策：方案 B — 自研虚拟表格**

- 不沿用 re-table（基于 el-table），需全新实现
- stk-table-vue 已验证架构可行，可参考其 `useFixedCol`、`useVirtual`、`useAreaSelection` 等
- 项目已有 VueUse、Element Plus，可复用工具与 UI 组件

### 2.2 设计原则

1. **虚拟滚动优先**：默认 `virtual: true`，适配万级行
2. **配置驱动**：`columns` JSON 配置为主，`render` / 插槽为辅
3. **Composable 架构**：每块能力独立 composable，主组件组装
4. **类型安全**：泛型 `RowData`，完整 TS 类型导出
5. **按需加载**：编辑、选区、列设置等按需挂载，减少首屏开销

## 四、核心类型定义（草案）

```ts
/** 泛型行数据 */
export type RowData = Record<string, any>

/** ──── 列配置 ──── */
export interface VirtualTableColumn<T = RowData> {
  key: string
  dataIndex?: keyof T & string
  title?: string
  width: number
  minWidth?: number
  maxWidth?: number
  fixed?: 'left' | 'right'
  align?: 'left' | 'center' | 'right'
  sortable?: boolean | 'custom'
  resizable?: boolean
  ellipsis?: boolean

  // 显隐
  hidden?: boolean

  // 编辑
  editable?: boolean | ((row: T) => boolean)
  component?: string | Component
  componentProps?: Record<string, any> | ((row: T, column: VirtualTableColumn<T>) => Record<string, any>)

  // 校验
  rules?: RuleItem | RuleItem[]

  // 渲染
  formatter?: (value: any, row: T, rowIndex: number) => any
  render?: (ctx: CellContext<T>) => VNode
  renderHeader?: (column: VirtualTableColumn<T>) => VNode

  [key: string]: any
}

/** 单元格上下文 */
export interface CellContext<T = RowData> {
  row: T
  column: VirtualTableColumn<T>
  value: any
  rowIndex: number
  colIndex: number
}

/** ──── 表格 Props ──── */
export interface VirtualTableProps<T = RowData> {
  data: T[]
  columns: VirtualTableColumn<T>[]
  rowKey?: string | ((row: T) => string | number)

  // 虚拟滚动
  virtual?: boolean
  virtualX?: boolean
  rowHeight?: number
  headerRowHeight?: number

  // 交互
  stripe?: boolean
  bordered?: boolean
  rowHover?: boolean
  cellActive?: boolean
  rowActive?: boolean
  areaSelection?: boolean

  // 编辑
  editable?: boolean | 'row' | 'cell' | 'manual'

  // 列
  columnSetting?: boolean
  colResizable?: boolean
  colDrag?: boolean | { mode?: 'insert' | 'swap' }

  // 行拖拽
  rowDrag?: boolean | { handle?: string } // handle 为拖拽把手选择器，默认整行可拖

  // 热键
  hotkeys?: HotkeyBinding[]
  hotkeyEnabled?: boolean

  // 分页
  pagination?: false | PaginationConfig

  // 表格自适应高度（容器填满剩余视口，供虚拟滚动计算可见区）
  adaptive?: boolean | AdaptiveConfig
}

/** 自适应高度配置 */
export interface AdaptiveConfig {
  /** 顶部偏移（px），如导航栏、工具栏高度 */
  offsetTop?: number
  /** 底部偏移（px），如分页器、页脚高度 */
  offsetBottom?: number
  /** 排除的兄弟选择器，计算高度时不计入 */
  excludeSelectors?: string[]
}
```

## 六、性能优化要点

| 场景 | 策略 |
|------|------|
| 自适应高度 | `useResizeObserver` 监听容器，动态设置高度，resize 节流；与 `use-virtual-scroll` 联动 |
| 虚拟滚动 | 只渲染可见行 + 少量 buffer，总 DOM 数恒定 |
| 列虚拟 | 列数 > 50 时考虑 virtualX |
| 响应式 | `shallowRef` 存大数组，避免深度响应式 |
| 编辑态 | 非编辑单元格不挂载组件，按需渲染 |
| 热键 | `when` 守卫短路，避免无效 context 构建 |
| 滚动 | `passive: true`、`requestAnimationFrame` 节流 |
| 选区 | 使用 CSS 绘制选区，避免大量 DOM |

## 七、技术决策记录

### D1: 自研 vs 基于 el-table-v2

**决策：自研虚拟表格**

- 不沿用 re-table（el-table 封装），需全新架构
- el-table-v2 API 与 el-table 不同，扩展编辑、选区等需深入改造
- 参考 stk-table-vue 已验证的虚拟滚动架构，可控性更高

### D2: 与 re-table 的关系

**决策：独立组件 `VirtualTable`**

- 不共用 re-table 代码
- 新目录 `virtual-table/`，新路由 `/virtual-table`
- 未来可逐步迁移业务后下线 re-table

### D3: 依赖

- **Vue 3**、**VueUse**、**Element Plus**（编辑器、Tooltip 等）
- **async-validator**（校验）
- 不引入 handsontable、vxe-table 等重型依赖
