# PlusTable 多级表头 — 列设置：仅顶层可拖拽 + 树状展示

> **选型**：参考业界列设置设计，采用**树状展示 + 仅顶层表头可拖拽**的形态。
>
> **实现选型（2025-03-06 讨论调整）**：使用 Element Plus **el-tree**（或 el-tree-v2）展示列树，其内置 `draggable` 支持节点拖拽，配合 `allow-drag` / `allow-drop` 可限制**仅顶层节点可拖拽**、仅允许放在顶层项之间。

---

## 一、形态定义

### 1.1 展示形态（树状）

| 要素 | 说明 |
|------|------|
| **层级结构** | 父列（如「基本信息」「组织信息」）可展开/折叠，子列缩进显示 |
| **顶层项** | 序号、头像、基本信息、名字、是否启用、评分… 等一级列 |
| **子列** | 如「基本信息」下：所在地、年龄、性别、邮箱；「组织信息」下：部门、团队 |
| **缩进** | 子列相对父列缩进，视觉上体现父子关系 |

### 1.2 拖拽形态（仅顶层）

| 要素 | 说明 |
|------|------|
| **拖拽** | 仅顶层项可拖拽（el-tree `allow-drag` 限制）；子列不可拖拽 |
| **拖拽效果** | 拖拽父列时，其下子列整组随父列移动 |
| **放置目标** | 仅允许放在其他顶层项之间；子列不参与拖拽目标 |

### 1.3 UI 元素（参考）

| 区域 | 元素 | 说明 |
|------|------|------|
| **顶部** | 「全部」复选框 | 全选/反选所有列显隐 |
| **列表项** | 复选框 | 控制该列/该组显隐 |
| **列表项** | 拖拽 | el-tree `draggable`，仅顶层项可拖拽（`allow-drag`） |
| **列表项** | 列名 | label 或 prop |
| **列表项** | 固定图标（左/右） | 可选，控制列 fixed |
| **底部** | 恢复默认、取消、确认 | 操作按钮 |

## 二、数据结构与 API

### 2.1 列设置数据源（el-tree 兼容格式）

```ts
/** 列设置树节点：供 el-tree 展示与拖拽使用 */
interface ColumnSettingNode {
  column: PlusTableColumn;
  /** 唯一标识：有 prop 用 prop，无 prop 用 _group:label 等 */
  id: string;
  /** el-tree 展示用 */
  label: string;
  depth: number;
  children?: ColumnSettingNode[];
  /** 是否顶层（可拖拽）；el-tree allow-drag 根据此字段限制 */
  isTopLevel: boolean;
}

/** 获取列设置树：保持 columns 树形，按 columnOrder 重排顶层顺序 */
function getColumnSettingTree(columns, columnOrder): ColumnSettingNode[]
```

### 2.2 顶层顺序与持久化

- `columnOrder`：存**顶层**列的 id 顺序（prop 或 `_group:xxx`）
- 拖拽仅改变顶层 order；子列顺序沿用 `initialColumns` 的树结构
- 持久化格式与现有一致：`{ prop, hidden, order }[]`，仅对顶层列写入 order

### 2.3 显隐逻辑

| 类型 | 勾选行为 |
|------|----------|
| 顶层叶子列 | 控制该列显隐 |
| 顶层父列 | 勾选 = 整组显示；取消 = 整组隐藏 |
| 子列 | 单独勾选控制该子列显隐；父列取消时整组隐藏，子列勾选无效 |

## 四、与现有实现的衔接

### 4.1 保留

- `flattenColumnsWithProp`：继续用于 navigableColumns、热键、编辑等
- 显隐 `hiddenColumns`、列宽 `columnWidths` 的持久化

### 4.2 调整

- `getOrderedColumnsWithProp` → 拆为：
  - `getColumnSettingTree()`：树形，供面板展示
  - 顶层 order 单独维护，供拖拽和 `visibleColumns` 排序
- `visibleColumns`：按顶层 order 重排顶层，再递归应用 `filterColumnsTree`（显隐）

### 4.3 可选

- 列宽输入框：可在树形项上保留，仅叶子列展示
- 固定：若需支持，在顶层项或叶子项上增加固定图标

### 4.4 el-tree 选型与实现要点

| 组件 | 适用场景 | 说明 |
|------|----------|------|
| **el-tree** | 推荐 | 列设置通常列数有限（< 100），无需虚拟滚动；API 成熟，`allow-drag` / `allow-drop` 满足仅顶层可拖拽需求 |
| **el-tree-v2** | 列数极多时 | 虚拟化树，适用于数千节点；若未来列数暴增可切换 |

**实现要点**：

- 树数据需含 `label`、`id`、`children`，与 `ColumnSettingNode` 对齐
- `allow-drag(node)`：返回 `node.data?.isTopLevel === true`
- `allow-drop(draggingNode, dropNode, type)`：`type` 为 `'prev' | 'next' | 'inner'`，仅允许 `prev`/`next` 且 `dropNode.data?.isTopLevel === true`
- `@node-drop`：从树根按深度优先遍历提取顶层 id 顺序，更新 `columnOrder` 并触发 `reorderColumns` 或等价逻辑
