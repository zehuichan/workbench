# ReTableNext — 基于 Element Plus el-table 的企业级增强组件

> **设计约束**：强制使用 Element Plus `el-table`，**不采用虚拟滚动**。  
> 定位：配置式驱动、类 Excel 交互、可编辑、全量渲染的表格组件。

| 约束项      | 说明                                       |

| **大数据**  | 依赖分页加载，不提供虚拟滚动能力                         |

## 二、技术选型与设计原则

### 2.1 基座选择

**决策：Element Plus `el-table`**

- 与项目栈一致，透传 el-table 原生 props/events/slots
- 不引入 el-table-v2（虚拟表格 API 不同，且与「取消虚拟滚动」约束冲突）
- 继承 re-table 架构，在其基础上增加 VirtualTable 规划中的能力

### 2.2 设计原则

1. **el-table 基座**：核心渲染由 el-table 完成，扩展层仅做封装与增强
2. **配置驱动**：`columns` JSON 配置为主，`render` / 插槽为辅
3. **Composable 架构**：每块能力独立 composable，主组件组装
4. **类型安全**：泛型 `RowData`，完整 TS 类型导出
5. **按需加载**：编辑、选区、列设置等按需挂载，减少首屏开销

## 四、核心类型定义（草案）

```ts
/** 泛型行数据 */
export type RowData = Record<string, any>

/** ──── 列配置 ──── */
export interface ReTableNextColumn<T = RowData> {
  key?: string
  prop?: keyof T & string
  label?: string
  width?: number | string
  minWidth?: number | string
  maxWidth?: number | string
  fixed?: 'left' | 'right'
  align?: 'left' | 'center' | 'right'
  sortable?: boolean | 'custom'
  // 显隐
  hidden?: boolean

  // 编辑
  editable?: boolean | ((row: T) => boolean)
  component?: string | Component
  componentProps?: Record<string, any> | ((row: T, column: ReTableNextColumn<T>) => Record<string, any>)

  // 校验
  rules?: RuleItem | RuleItem[]

  // 渲染
  formatter?: (value: any, row: T, rowIndex: number) => any
  render?: (ctx: CellContext<T>) => VNode
  renderHeader?: (column: ReTableNextColumn<T>) => VNode

  [key: string]: any
}

/** 单元格上下文 */
export interface CellContext<T = RowData> {
  row: T
  column: ReTableNextColumn<T>
  value: any
  rowIndex: number
  colIndex: number
}

/** ──── 表格 Props ──── */
export interface ReTableNextProps<T = RowData> {
  data: T[]
  columns: ReTableNextColumn<T>[]
  rowKey?: string | ((row: T) => string | number)

  // 交互（无 virtual 相关）
  stripe?: boolean
  border?: boolean
  rowHover?: boolean
  cellActive?: boolean
  rowActive?: boolean
  areaSelection?: boolean

  // 编辑
  editable?: boolean | 'row' | 'cell' | 'manual'

  // 列（列宽拖动使用 el-table 原生支持，不单独实现）
  columnSetting?: boolean
  colDrag?: boolean | { mode?: 'insert' | 'swap' }

  // 行拖拽
  rowDrag?: boolean | { handle?: string }  // handle 为拖拽把手选择器，默认整行可拖

  // 热键
  hotkeys?: HotkeyBinding[]
  hotkeyEnabled?: boolean

  // 分页
  pagination?: false | PaginationConfig

  // 表格自适应高度（需配合 max-height 使用，填满剩余视口）
  adaptive?: boolean | AdaptiveConfig
}

/** 自适应高度配置 */
export interface AdaptiveConfig {
  /** 顶部偏移（px），如导航栏、工具栏高度 */
  offsetTop?: number
  /** 底部偏移（px），如分页器、页脚高度 */
  offsetBottom?: number
  /** 排除的兄弟选择器，计算高度时不计入 */
  excludeSelectors?: string[]
}
```

| 区域         | 结构                                          | 插槽                             | 说明                              |

自适应高度计算时，`offsetTop`/`offsetBottom` 需考虑 header/footer 实际高度（或由使用方在 AdaptiveConfig 中指定）。

### 阶段 1：配置式列 + 列渲染架构（Week 2）

> **目标（以当前代码为准）**：基于 `el-table` + `el-table-column` 实现配置式列驱动；列渲染由 **SFC `<template>` 的 `ReTableNextColumn.vue`** 完成，支持 `render`、插槽、多级表头、特殊列（selection/index/expand）。可选演进方向：TSX 列组件、`useColumnBind` 集成、表级默认列绑定。

#### 当前实现（以代码为准）

| 维度 | 实现方式 | 代码位置 |
|------|----------|----------|
| **列渲染组件** | `re-table-next-column.vue`，SFC `<template>` + `v-if/v-else-if` 分支 | `re-table-next-column.vue` |
| **主组件列分支** | 特殊列（selection/index/expand）用 `h(ElTableColumn, column, slots)` 直接渲染；其余列 `<ReTableNextColumn :item="column" v-slot="slots" />` | `re-table-next.vue` default slot |
| **列绑定** | 直接 `v-bind="item"` 到 `el-table-column`，未使用 `useColumnBind` | `re-table-next-column.vue` |
| **默认列属性** | 主组件 `showOverflowTooltip` 等透传到 `el-table`，未做列级 `provide/inject` | `re-table-next.vue`、`constants.ts` 无 `DEFAULT_COLUMN_BIND_KEY` |
| **单元格渲染优先级** | 1. `children` → 递归 `<ReTableNextColumn>`；2. 插槽 `#cell-${prop}`；3. `item.render` 经 `Renderer`（当前仅传 `scope.row`）；4. 原始值 `get(scope.row, item.prop)` | `re-table-next-column.vue` template #default |
| **表头** | 仅透传插槽 `#header-${prop}`，无插槽时为空（未实现 `label` / `renderHeader` 回退） | `re-table-next-column.vue` template #header |
| **多级表头** | 有 `children` 时在 default 内递归 `ReTableNextColumn`，插槽通过 `$slots` 透传 | 同上 |
| **expand** | 主组件内单独分支，`h(ElTableColumn, column, { default: scope => slots.expand?.(scope) })` | `re-table-next.vue` |

#### 渲染器与工具（以代码为准）

| 文件 | 作用 | 是否被列组件使用 |
|------|------|------------------|
| `renderers/renderer.tsx` | `Renderer` 组件：接收 `render` + `data`，执行 `render(data)` | 是，列组件传 `:data="scope.row"`，即 **render 当前仅接收 row**，非完整 `CellContext` |
| `renderers/cell-renderer.ts` | `getCellValue(row, column)` 取单元格值 | 否 |
| `renderers/header-renderer.ts` | `renderHeader(column)`：`renderHeader` 函数优先，否则 `label` | 否 |
| `composables/use-column-bind.ts` | 过滤自定义属性，返回可展开到 `ElTableColumn` 的绑定 | 已实现并导出，**未在列/主组件中接入** |

## 六、性能优化要点（非虚拟滚动场景）

| 场景    | 策略                                                            |
|-------|---------------------------------------------------------------|
| 自适应高度 | `useResizeObserver` 监听容器，动态设置 el-table `max-height`，resize 节流 |
| 大数据   | 分页加载，全量渲染千级以内数据                                               |
| 响应式   | `shallowRef` 存大数组，避免深度响应式                                     |
| 编辑态   | 非编辑单元格不挂载组件，按需渲染                                              |
| 热键    | `when` 守卫短路，避免无效 context 构建                                   |
| 选区    | 使用 CSS 绘制选区，避免大量 DOM                                          |

## 七、技术决策记录

### D1: 基座 — el-table vs el-table-v2 vs 自研

**决策：el-table**

- 用户明确要求取消虚拟滚动，el-table-v2 为虚拟表格，不符合
- 自研虚拟表格与「取消虚拟滚动」约束冲突
- el-table 成熟稳定，与 Element Plus 生态一致

### D2: 与 re-table / VirtualTable 的关系

**决策：独立组件 `ReTableNext`**

- 新目录 `re-table-next/`，新路由 `/re-table-next`
- 可参考 re-table 的 el-table 封装经验
- 吸收 VirtualTable 规划中的选区、剪贴板、列拖拽等能力
- 不包含虚拟滚动

### D3: 依赖

- **Vue 3**、**VueUse**、**Element Plus**（el-table、编辑器、Tooltip 等）
- **async-validator**（校验）
- 不引入 handsontable、vxe-table 等重型依赖
