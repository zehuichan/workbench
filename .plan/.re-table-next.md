# ReTableNext — 基于 Element Plus el-table 的企业级增强组件

> **设计约束**：强制使用 Element Plus `el-table`，**不采用虚拟滚动**。  
> 定位：配置式驱动、类 Excel 交互、可编辑、全量渲染的表格组件。

## 二、技术选型与设计原则

### 2.1 基座选择

**决策：Element Plus `el-table`**

- 与项目栈一致，透传 el-table 原生 props/events/slots
- 不引入 el-table-v2（虚拟表格 API 不同，且与「取消虚拟滚动」约束冲突）
- 继承 re-table 架构，在其基础上增加 VirtualTable 规划中的能力

### 2.2 设计原则

1. **el-table 基座**：核心渲染由 el-table 完成，扩展层仅做封装与增强
2. **配置驱动**：`columns` JSON 配置为主，`render` / 插槽为辅
3. **Composable 架构**：每块能力独立 composable，主组件组装
4. **类型安全**：泛型 `RowData`，完整 TS 类型导出
5. **按需加载**：编辑、列设置等按需挂载，减少首屏开销

## 四、核心类型定义（草案）

```ts
/** 泛型行数据 */
export type RowData = Record<string, any>

/** ──── 列配置（继承 Element Plus TableColumnCtx，仅声明扩展项）──── */
import type { TableColumnCtx } from 'element-plus'

export interface ReTableNextColumn<T = RowData>
  extends Partial<Omit<TableColumnCtx<T>, 'children'>> {
  hidden?: boolean
  editable?: boolean | ((row: T) => boolean)
  component?: string | Component
  componentProps?: Record<string, any> | ((row: T, column: ReTableNextColumn<T>) => Record<string, any>)
  rules?: RuleItem | RuleItem[]
  render?: (ctx: CellContext<T>) => VNode
  children?: ReTableNextColumn<T>[]
}

/** 单元格上下文 */
export interface CellContext<T = RowData> {
  row: T
  column: ReTableNextColumn<T>
  value: any
  rowIndex: number
  colIndex: number
}

/** ──── 表格 Props ──── */
export interface ReTableNextProps<T = RowData> {
  data: T[]
  columns: ReTableNextColumn<T>[]
  rowKey?: string | ((row: T) => string | number)

  // 交互（无 virtual 相关）
  stripe?: boolean
  border?: boolean
  rowHover?: boolean
  cellActive?: boolean
  rowActive?: boolean

  // 编辑
  editable?: boolean | 'row' | 'cell' | 'manual'

  // 列（列宽拖动使用 el-table 原生支持，不单独实现）
  columnSetting?: boolean

  // 热键
  hotkeys?: HotkeyBinding[]
  hotkeyEnabled?: boolean

  // 分页
  pagination?: false | PaginationConfig

  // 表格自适应高度（需配合 max-height 使用，填满剩余视口）
  adaptive?: boolean | AdaptiveConfig
}

/** 自适应高度配置 */
export interface AdaptiveConfig {
  /** 顶部偏移（px），如导航栏、工具栏高度 */
  offsetTop?: number
  /** 底部偏移（px），如分页器、页脚高度 */
  offsetBottom?: number
  /** 排除的兄弟选择器，计算高度时不计入 */
  excludeSelectors?: string[]
}
```

> **目标（以当前代码为准）**：基于 `el-table` + `el-table-column` 实现配置式列驱动；列渲染由 **SFC `<template>` 的 `ReTableNextColumn.vue`** 完成，支持 `render`、插槽、多级表头、特殊列（selection/index/expand）。可选演进方向：TSX 列组件。

#### 当前实现（以代码为准）

| 维度           | 实现方式                                                                                                                                                                                                                                                  | 代码位置                                                           |
|--------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------|
| **列渲染组件**    | `re-table-next-column.vue`（列级包裹）+ `re-table-next-cell.vue`（单元格渲染）。Column 负责 `el-table-column` 绑定、表头、多级表头递归；Cell 负责编辑/展示分支 | `re-table-next-column.vue` + `re-table-next-cell.vue` |
| **主组件列分支**   | 特殊列（selection/index/expand）用 `h(ElTableColumn, column, slots)` 直接渲染；其余列 `<ReTableNextColumn :item="column" v-slot="slots" />`                                                                                                                         | `re-table-next.vue` default slot                               |
| **列绑定**      | `columnBindings` computed 过滤 `hidden/editable/component/componentProps/rules/render/children`，`v-bind` 到 `el-table-column`                                                                                                                               | `re-table-next-column.vue`                                     |

#### 四种编辑模式

| `editable` 值 | 内部模式 | 触发方式 | 编辑范围 | Tab 行为 |
|:---:|:---:|---|---|---|
| `false` | `'none'` | 禁用 | 无 | 仅导航 |
| `true` / `'cell'` | `'cell'` | F2 / Enter / 双击 / 直接输入 | 单个单元格 | 确认+右移 |
| `'row'` | `'row'` | F2 / Enter / 双击 / 直接输入 | 整行所有可编辑列 | 行内移动（不确认） |
| `'manual'` | `'manual'` | 仅 `startEdit()` API | 单个单元格 | 确认+右移 |

**manual 模式使用场景**（编辑仅由业务/代码通过 `startEdit()` 触发，不响应双击/F2/Enter/直接输入）：

| 场景 | 说明 |
|------|------|
| 权限/审批后开放编辑 | 表格默认只读，仅在有编辑权限或审批通过后显示「开始编辑」等入口，由业务决定何时调用 `startEdit()`。 |
| 先选范围再编辑 | 用户先勾选行或选中区域，再点「编辑选中项」；代码调用 `startEdit(rowIndex, colProp)` 只对选中格进入编辑，减少误触。 |
| 主从/向导中的编辑阶段 | 主表只读，选中某行进入「详情/编辑」步骤后再允许编辑；父组件在切换步骤或打开编辑面板时调用 `startEdit()`。 |
| 审计/合规需明确进入编辑 | 需记录「谁在何时点击了编辑」；仅「开始编辑」按钮触发 `startEdit()`，便于在 `@click` 打点或写审计日志。 |
| 触屏/大屏/Kiosk 防误触 | 不希望单击/双击即进入编辑；仅单独「编辑」按钮进入编辑，减少误操作。 |
| 业务解锁后才可编辑 | 如订单未支付时只读，支付后「解锁」再允许编辑；解锁时由代码调用 `startEdit()`。 |
| 与弹窗/侧栏联动 | 打开右侧抽屉或弹窗时允许编辑；弹窗 `open` 时 `startEdit()`，关闭时 `cancelEdit()`。 |
| 只读为主 + 行内「编辑」按钮 | 表格以查阅为主，每行或表头有「编辑」按钮，点击后对该行/格调用 `startEdit(rowIndex, colIndex)`。 |

Demo：`/re-table-next/stage3` 选择 manual 模式，演示工具栏「开始编辑」与行内「编辑名称」「编辑备注」两种用法。

- `editMode` computed 由 `resolveEditMode(editable)` 统一解析
- `autoTriggerEnabled` computed：`cell`/`row` 模式为 `true`，`manual`/`none` 为 `false`
- Row 模式使用 `editingValues: Record<string, any>` 存储整行编辑值，`confirmEdit` 返回 `EditChangeRecord[]`
- 编辑历史支持分组：row 模式一次确认推入一组 `EditChangeRecord[]`，整组原子撤销/重做

#### 自定义编辑器插槽

Cell 组件（`re-table-next-cell.vue`）支持 `#editor-${column.prop}` 插槽，优先级高于 `column.component` 配置。插槽由主组件经 Column → Cell 逐级透传。插槽 scope 通过 `editorSlotScope` computed 构造，包含 `modelValue` + `onUpdate:modelValue`，使消费侧 `v-bind="scope"` 即可完成 v-model 绑定：

```ts
{
  row: RowData,                        // 行数据
  column: ReTableNextColumn,           // 列配置
  modelValue: any,                     // 当前编辑值（v-bind 自动映射为 :model-value）
  'onUpdate:modelValue': (val) => void, // 更新编辑值（v-bind 自动映射为 @update:model-value）
  confirm: () => void,                 // 确认编辑
  cancel: () => void,                  // 取消编辑
}
```

使用示例：

```vue
<template #editor-remark="scope">
  <el-input v-bind="scope" placeholder="请输入备注…" />
</template>
```

## 七、技术决策记录

### D1: 基座 — el-table vs el-table-v2 vs 自研

**决策：el-table**

- 用户明确要求取消虚拟滚动，el-table-v2 为虚拟表格，不符合
- 自研虚拟表格与「取消虚拟滚动」约束冲突
- el-table 成熟稳定，与 Element Plus 生态一致

### D2: 与 re-table / VirtualTable 的关系

**决策：独立组件 `ReTableNext`**

- 新目录 `re-table-next/`，新路由 `/re-table-next`
- 可参考 re-table 的 el-table 封装经验
- 吸收 VirtualTable 规划中的编辑、校验、行/列操作等能力
- 不包含虚拟滚动

### D3: 依赖

- **Vue 3**、**VueUse**、**Element Plus**（el-table、编辑器、Tooltip 等）
- **async-validator**（校验）
- 不引入 handsontable、vxe-table 等重型依赖
