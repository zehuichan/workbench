# 项目复盘 Workflow

基于 `.plan/` 下规划文档进行**阶段性复盘**，检测任务清单有无完成，避免 context 过长导致 agent 降智。

## 复盘步骤

1. **任务清单核对**：逐条检查 `- [ ]` 是否应改为 `- [x]`（已完成则勾选）
2. **里程碑更新**：同步更新「〇、进度里程碑」表
   - 各阶段完成度：已完成数 / 总任务数
   - 当前进度：下一批待办简述
   - 整体完成度：总已完成 / 总任务
3. **Demo 验收**：各阶段 Demo 路由是否可访问、功能是否符合预期
4. **偏差说明**：若有任务取消、拆分或新增，在规划文档中注明

请逐步执行每个步骤，完成后汇报结果。
